# Architecture notes for slides

## Hosted/cloud-native design

User browser -> Azure Static Web Apps -> React static assets.

React calls `/api/*` endpoints. Azure Static Web Apps provides managed Azure Functions for the REST API. The API writes images to Azure Blob Storage and metadata/comments/ratings to Azure Cosmos DB for NoSQL.

Authentication is handled by Azure Static Web Apps built-in auth. All signed-in users receive the `authenticated` role. Creator users are manually invited in Azure Static Web Apps Role Management with the custom `creator` role. The upload API also checks this role before accepting content.

## Scalable elements

- Static Web Apps: CDN-style static hosting and automated deployment from GitHub.
- Azure Functions: serverless HTTP endpoints that scale with demand.
- Blob Storage: object storage for large unstructured image files.
- Cosmos DB: cloud NoSQL database for metadata and user interactions.
- Role-based access: separates viewer and creator capabilities.
- GitHub Actions: CI/CD pipeline generated by Azure Static Web Apps.

## Containerised alternative to discuss

A containerised option would package the frontend/backend into Docker containers and deploy through Azure Container Apps or Azure Kubernetes Service. This gives more runtime control but adds operational complexity. For this coursework, the hosted/serverless option is simpler and demonstrates cloud-native scalability with less management overhead.
